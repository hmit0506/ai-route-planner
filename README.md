## 项目背景

本地生活路线规划存在三类典型痛点：

1. **能去但不好用**：只给店铺列表，路线顺路性差、跨区折返，无法直接执行
2. **看得到店，看不到风险**：缺少排队、踩雷、时段等来自真实消费者的体验信号
3. **个性化落地难**：用户一句自然语言（预算、口味、场景、天气）难被完整理解并转化为路线

**途晓**以**香港**为核心数据场景——18,248 条覆盖全港主要商圈（旺角、中環、尖沙咀、銅鑼灣等）的真实本地 POI，来源 OpenRice 五年真实评论数据；同时支持其他主要城市的实时搜索。目标是生成**可执行、可解释、可导航、可复用**的本地路线方案，面向「现在就出发」的高频本地生活场景。

## 核心创新

### 香港真实评论信号驱动路线决策
与仅靠地图 API 元数据的方案不同，我们对 OpenRice 香港 2021–2025 年五年真实评论进行了系统化处理，将排队风险、踩雷概率、打卡热度、本地特色、场景适配等评论语义转为可计算的信号指标，**直接参与路线决策和候选排序**——近年少评论的店铺自动降权，高风险店铺提前过滤，而非仅在页面展示标签。

> 处理规模：23,541 个原始香港 POI，质量过滤后 18,248 条入库，跨越五年评论确保活跃度可追踪。

### 多智能体协同 + 自动校验兜底
全流程由多个专职智能体接力完成：意图解析、天气感知、候选召回、地理聚合、路线决策、数据补充、结果输出各司其职。路线生成后系统自动校验站点数量、餐饮安排、类别覆盖是否合理，不通过则附带原因说明重试，最终兜底保证输出合规——将 AI 的灵活性与规则引擎的稳定性相结合。

### 天气 × 地理 × 评论信号三维融合
三者同时作用于路线决策：实时天气预报识别晴雨高温，自动调整室内/户外偏好；90 余个香港社区地理坐标锚定出行范围，避免跨区折返；评论信号将高风险、低活跃场所在排序层提前降权——三维约束缺一不可。

## 功能全景

### 读懂你说的话
- **自然语言输入**：直接说"旺角下午、預算400、想吃日本料理"，无需结构化填表
- **三语界面**：支持繁体中文、简体中文及英文，全部提示、摘要、分类名称随语言完整切换；亦可支撑美团未来的出海业务，面向香港、海外华人及国际用户无缝切换
- **习惯用词对齐**：理解"壽司"、"下午茶"、"打邊爐"等本地说法，精准匹配数据库

### 香港本地深度数据
- **18,248 条香港真实 POI**：18,075 家餐厅 + 173 个文化/娱乐/自然景点，覆盖 75 个本地饮食类别，88% 餐厅带多标签
- **五年评论信号**：排队风险、踩雷概率、拍照热度、本地特色、场景适配等多维信号直接参与推荐决策
- **实时兜底**：本地候选不足时自动调用地图搜索补充，其他城市同样支持

### 路线规划
- **区域聚焦**：所有站点锁定在步行可达范围内，不跨区折返
- **天气感知**：获取出行日期实时天气，雨天/高温自动切换室内优先策略，并推送天气提示
- **多维决策**：综合评分、性价比、排队高峰/非高峰、口味卫生环境评分、销量热度、团购优惠
- **营业过滤**：自动排除与出行时段不重叠的场所

### 展示与分享
- **地图可视化**：所有站点在地图上打点，叠加真实步行路径蓝线
- **交互式地图**：前端可缩放，点击站点弹出详情
- **一键导航**：每个站点附带高德导航链接，点击即可跳转导航
- **小红书式攻略**：路线发出后异步生成社媒风格图文，含亮点、避坑、话题标签，繁简英三语各有专属风格

### 对话与记忆
- **局部替换**：说"换一家不排队的餐厅"即可，无需重新规划整条路线
- **用户记忆**：记住菜系偏好、忌口、消费习惯，已去过的地方自动排除
- **出行报告**：每次规划后说明哪些需求已满足、哪些做了替代安排，透明可信

## 系统架构

### 首次生成（2 次 AI 推理）

```
用户输入
  → 意图解析      AI①：理解需求，提取城市/时段/预算/口味/场景等结构化信息
  → 天气感知      获取出行日期实时天气，注入路线决策
  → 候选召回      香港本地数据库优先；不足时实时搜索补充
  → 地理聚合      锁定步行范围，按时长确定站点数
  → 路线决策      AI②：多维权衡，天气感知，自动校验 + 兜底重试
  → 数据补充      排队/团购/趋势/标签全量填充
  → 结果输出      步行路径、导航链接、地图、摘要、小红书攻略
```

### 局部替换（1 次 AI 推理）

```
用户："换一家不排队的餐厅"
  → 替换解析      AI①：定位替换目标，提取约束条件
  → 定向召回      只搜被替换类别的候选
  → 最优替换      按约束选出最优 POI，合并回原路线
  → 数据补充 → 结果输出
```

技术实现细节见[系统架构文档](docs/ARCHITECTURE.md)，前端接入见[前端接入指南](docs/FRONTEND.md)。

## 快速开始（本地开发）

### 1. 一键建环境

需要 Python 3.11+（推荐 3.12）。

```bash
bash setup.sh
```

脚本自动完成：创建 `.venv`、安装依赖、生成 `.env.example`。数据库在首次启动时自动从 CSV 生成。

### 2. 填入 API Key

编辑 `.env`：

| 变量 | 用途 | 必填 |
|---|---|---|
| `DEEPSEEK_API_KEY` | 主力 LLM | ✅ |
| `ANTHROPIC_API_KEY` | 备用 LLM | 建议 |
| `AMAP_API_KEY` | 高德地图（静态地图 + 天气 + 步行路径） | ✅ |

> 前端动态地图需单独申请高德 Web 端 JS Key，详见[前端接入指南](docs/FRONTEND.md)。

### 3. 启动后端

```bash
PYTHONPATH=. .venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

访问 `http://localhost:8000/health` 返回 `{"status":"ok"}` 即成功。

### 4. 验证流水线

```bash
PYTHONPATH=. .venv/bin/python3 scripts/run_pipeline.py
PYTHONPATH=. .venv/bin/python3 scripts/run_pipeline.py "旺角附近下午，想吃日本料理，預算400港幣"
PYTHONPATH=. .venv/bin/python3 scripts/run_pipeline.py "中環一整天，包括午餐和晚餐，預算600"
```
